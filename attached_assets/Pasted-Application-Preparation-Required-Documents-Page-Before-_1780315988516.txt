Multi-Step Application Validation Rules
Create a strict multi-step application workflow.
Mandatory Completion Requirement
Applicants may NOT move to the next page until the current page has been fully completed.
Every required field must pass validation before the "Next" button becomes active.
Examples:
Missing required fields = cannot continue
Missing uploads = cannot continue
Missing signatures = cannot continue
Missing acknowledgments = cannot continue
Missing photos = cannot continue
Display validation messages directly below each missing field.

Admin Exception Rules
IMPORTANT:
Only Admin users shall have unrestricted navigation privileges.
Admin users should be able to:
Open any application page
Skip pages
Navigate forward
Navigate backward
Review incomplete applications
Edit application records
Request corrections
Applicants, Providers, and Members may NOT bypass validation requirements.
Only Admin accounts may view or navigate incomplete sections without completing them.
Final Submission Lock
Once the application has been submitted:
Lock the application from editing.
Change status to:
Submitted
Pending Review
Applicants may only edit the application again if Admin changes the status to:
Additional Information Requested
At that point, unlock only the sections requiring correction.
Final Requirement
Create a professional, secure, step-by-step onboarding experience that ensures complete applications, improves approval quality, reduces fraud, and prevents applicants from skipping required sections while preserving full administrative review privileges for NFGN administrators.
